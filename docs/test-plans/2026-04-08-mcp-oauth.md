# MCP OAuth Bearer Token Auth — Human Test Plan

## Prerequisites

- Both `ultrabridge` and `ub-mcp` binaries built and deployed on the test server (192.168.9.52)
- `UB_DB_PATH` set to a valid notedb SQLite path, shared between both processes
- `ub-mcp` started with `--http :8081` and accessible from the internet (or via tunnel for claude.ai testing)
- Automated tests passing: `go test ./internal/mcpauth/ ./cmd/ub-mcp/ ./internal/web/`

## Phase 1: Startup Warning Behavior (AC2.5)

| Step | Action | Expected |
|------|--------|----------|
| 1 | Stop ub-mcp if running | Process stopped |
| 2 | Unset `UB_DB_PATH`, `UB_MCP_AUTH_TOKEN`, `UB_MCP_API_USER`, and `UB_MCP_API_PASS` | Variables cleared |
| 3 | Run `ub-mcp --http :8081` and observe stdout/stderr | Output contains `WARNING: no auth configured` |
| 4 | Stop ub-mcp | Process stopped |
| 5 | Set `UB_DB_PATH` to the valid notedb path | Variable set |
| 6 | Run `ub-mcp --http :8081` and observe stdout/stderr | Warning does NOT appear. Log contains `auth enabled` |

## Phase 2: Web UI Token Lifecycle (AC3.5)

| Step | Action | Expected |
|------|--------|----------|
| 1 | Open browser to `https://<host>:8443/settings` | Settings page loads, MCP Tokens card visible |
| 2 | Enter "browser-test-1" in the token label field and click Create | Page redirects. Green flash box appears with raw token (43 chars). Copy this token. |
| 3 | Navigate to any other tab (e.g., Files) | Page changes to Files tab |
| 4 | Navigate back to `/settings` (click Settings tab, or enter URL directly without `?new_token=` param) | Settings page loads. Green flash box is NOT visible. Token "browser-test-1" appears in token list with hash prefix, creation date, and "Never" for last used. |
| 5 | Click browser Back button repeatedly | Raw token flash box should not reappear |

## Phase 3: Cross-Component Token Revocation (AC3.6)

| Step | Action | Expected |
|------|--------|----------|
| 1 | Ensure both `ultrabridge` and `ub-mcp` are running, both pointing at the same `UB_DB_PATH` | Both processes running |
| 2 | Open `https://<host>:8443/settings` in a browser | Settings page loads with MCP Tokens card |
| 3 | Create a new token with label "revocation-test". Copy the displayed raw token. | Token created and displayed |
| 4 | In a terminal, run: `curl -s -o /dev/null -w "%{http_code}" -H "Authorization: Bearer <token>" http://localhost:8081/sse` | Returns `200` (or establishes SSE connection) |
| 5 | In the browser, find "revocation-test" in the token list and click the Revoke button | Token removed from list. Page redirects to `/settings#mcp-tokens`. |
| 6 | Repeat the curl command from step 4 with the same token | Returns `401` |

## End-to-End: claude.ai MCP Connectivity (AC4.1)

| Step | Action | Expected |
|------|--------|----------|
| 1 | Ensure `ub-mcp --http :8081` is running and reachable from the internet | Process accessible at public URL |
| 2 | Open `https://<host>:8443/settings` and create a token with label "claude-ai-test". Copy the raw token. | Token created |
| 3 | In claude.ai, configure a remote MCP server: URL = public ub-mcp endpoint, `authorization_token` = the copied raw token | MCP server configured |
| 4 | Start a new conversation in claude.ai. Ask it to search your notes. | Claude invokes `search_notes` tool. Results appear (or "no results" — either proves the tool call succeeded). |
| 5 | Return to UltraBridge settings page. Check "Last Used" column for "claude-ai-test". | Shows recent timestamp (within last few minutes) |
| 6 | Click Revoke on the "claude-ai-test" token | Token removed from list |
| 7 | Return to claude.ai and attempt another tool call | Tool call fails with authentication error |
