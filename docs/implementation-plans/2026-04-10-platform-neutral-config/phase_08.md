# Platform-Neutral Configuration Implementation Plan

**Goal:** Reduce install.sh to 3 prompts (port, username, password), update rebuild.sh, remove Supernote-centric naming from env vars, handler code, and documentation.

**Architecture:** install.sh shrinks drastically — all configuration beyond bootstrap moves to the Settings UI (Phase 7 setup mode handles first-boot credential setup). The compose file contains only bootstrap env vars. Platform-neutral naming replaces Supernote-specific references across the codebase: `snNotesPath` handler field, `DEFAULT_SUPERNOTE_DIR`, hardcoded `/mnt/supernote` paths, and README documentation.

**Tech Stack:** Bash, Docker Compose, Go (handler field renames)

### Task 1: Simplify install.sh to 3 prompts

**Verifies:** platform-neutral-config.AC3.1, platform-neutral-config.AC3.2, platform-neutral-config.AC4.3

**Files:**
- Modify: `install.sh` (major rewrite — reduce from 728 lines to ~150)

**Implementation:**

Rewrite install.sh to:

1. **Pre-flight checks:** Docker + Docker Compose (keep existing logic from lines 142-164)
2. **3 prompts:**
   - Port to expose (default: 8443)
   - Username (for initial admin)
   - Password (with confirmation)
3. **Build Docker image** (keep existing `docker build` logic)
4. **Generate minimal compose file:**

```yaml
services:
  ultrabridge:
    image: ultrabridge:latest
    build: .
    ports:
      - "${PORT}:8443"
    environment:
      - UB_DB_PATH=/data/ultrabridge.db
      - UB_LISTEN_ADDR=:8443
      - UB_TASK_DB_PATH=/data/ultrabridge-tasks.db
    volumes:
      - ./ultrabridge-data:/data
    restart: unless-stopped
```

No `.ultrabridge.env` file generated. No `.ub_password_hash` file. Bootstrap env vars are inline in the compose file.

5. **Start container, health check, print URL:**
```
UltraBridge is running at http://localhost:${PORT}
Open in your browser to complete setup.
```

6. **Seed initial credentials:** After container is healthy, POST to the setup endpoint or use a seed command to store the username + bcrypt-hashed password in the settings table. This could be done via:
   - `docker exec ultrabridge /ultrabridge seed-user --username "$USERNAME" --password "$PASSWORD"` (new subcommand)
   - Or via HTTP POST to `/setup/save` (if the container is reachable)

The simplest approach: add a `seed-user` subcommand to the ultrabridge binary that takes username and password, hashes the password, and writes both to the settings table via `notedb.SetSetting`. This avoids depending on HTTP being reachable and is consistent with the existing `hash-password` subcommand pattern.

**Unattended mode (`-y`):** Reads `UB_USERNAME`, `UB_PASSWORD`, `UB_PORT` from env vars and skips prompts (keep existing `-y` flag pattern).

**What's removed:**
- All 22+ configuration prompts beyond port/username/password
- `.ultrabridge.env` generation (all config moves to Settings UI)
- `.ub_password_hash` file generation
- `DEFAULT_SUPERNOTE_DIR` variable
- SPC auto-detection logic
- All Supernote-specific prompt text
- MariaDB/`.dbenv` handling (users configure SPC integration via Settings UI)

### Task 2: Add `seed-user` subcommand to ultrabridge binary

**Verifies:** platform-neutral-config.AC3.1

**Files:**
- Modify: `cmd/ultrabridge/main.go` (add subcommand handling for `seed-user`)

**Implementation:**

Add a `seed-user` subcommand alongside the existing `hash-password` subcommand. It takes username and password as arguments, hashes the password with bcrypt, and writes both to the settings table.

```go
case "seed-user":
    if len(os.Args) < 4 {
        fmt.Fprintln(os.Stderr, "usage: ultrabridge seed-user <username> <password>")
        os.Exit(1)
    }
    username, password := os.Args[2], os.Args[3]
    dbPath := envOrDefault("UB_DB_PATH", "/data/ultrabridge.db")
    db, err := notedb.Open(context.Background(), dbPath)
    if err != nil {
        fmt.Fprintf(os.Stderr, "open db: %v\n", err)
        os.Exit(1)
    }
    defer db.Close()
    hash, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
    if err != nil {
        fmt.Fprintf(os.Stderr, "hash password: %v\n", err)
        os.Exit(1)
    }
    ctx := context.Background()
    notedb.SetSetting(ctx, db, appconfig.KeyUsername, username)
    notedb.SetSetting(ctx, db, appconfig.KeyPasswordHash, string(hash))
    fmt.Println("User credentials saved.")
```
